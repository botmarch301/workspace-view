# 화면 구성 설계 (UI Layout)
> **Status**: 초안 | **Created**: 2026-03-29
> **참조**: [concept.md](concept.md) · [scenario.md](scenario.md) · [scenes.md](scenes.md)

---

## 설계 원칙

1. **수치 비노출** — 내부 수치(양기, 공포, 호감, 성욕, 흥분도)를 UI에 직접 표시하지 않는다
2. **몰입 우선** — HUD를 최소화하고, 필요한 정보는 환경/연출로 전달한다
3. **모드 전환 자연스럽게** — SIM ↔ VN ↔ ARC ↔ SEX 전환 시 UI가 부드럽게 변형된다

## 1. 공통 레이아웃

### 1-1. 해상도 및 비율
- **기본 해상도**: 1920×1080 (16:9)
- **세이프 존**: 좌우 상하 5% 마진
- **세로형 CG**: 캐릭터 스탠딩은 화면 중앙~우측 배치

### 1-2. 공통 UI 요소
| 요소 | 위치 | 설명 |
|------|------|------|
| **시스템 메뉴** | 우상단 (아이콘) | 세이브/로드/설정/종료. 평소 숨김, 마우스 호버 또는 ESC로 표시 |
| **퀵세이브** | 없음 | 의도적 생략 — 선택의 무게를 유지 |
| **오토세이브** | 매 행동 종료 시 | 슬롯 3개 로테이션 |

## 2. 일상 시뮬레이션 화면 (SIM)

> 하루의 행동을 선택하고 관리하는 메인 루프 화면

### 2-1. 레이아웃
```
┌─────────────────────────────────────────────────┐
│                                                 │
│              [배경: 집 뷰 A/B/C]                 │
│              (시간대에 따라 조명 변화)              │
│                                                 │
│                                                 │
│  ┌──────────┐                                   │
│  │ 시간 표시 │                                   │
│  │ ○월 ○일  │                                   │
│  │ [아침]   │                                   │
│  └──────────┘                                   │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │         행동 선택 메뉴 (하단)              │    │
│  │  [식사] [세신] [청소] [출근] [쇼핑]        │    │
│  │  [핸드폰] [VR] [수면] [인벤토리]          │    │
│  └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
```

### 2-2. 시간 표시
- **위치**: 좌상단
- **표시**: 날짜 + 시간대 (아침/낮/저녁/밤/심야)
- **형식**: 달력 느낌 없이, 간결한 텍스트
- **하늘 표현**: 배경 하늘에 상시 표시 (HUD가 아닌 배경 요소)
  - 낮: 해, 구름 (시간대에 따라 위치·색상 변화)
  - 밤: 달 (월상이 자연스럽게 변화, 보름달 접근을 체감)

### 2-3. 행동 선택 메뉴
- **위치**: 하단 중앙
- **표시 방식**: 아이콘 + 텍스트 버튼
- **비활성 표시**: 시간대/조건에 따라 일부 버튼 흐리게 (출근은 밤에 비활성 등)
- **해금 시스템**: 
  - 1막: 식사, 세신, 출근, 쇼핑, 핸드폰, VR, 수면, 인벤토리
  - 2막: (동일)
  - 3막: + 인벤토리에 아이템 배치 기능 추가 (제물 등)
  - 4막: + 스킨십, 성감 개발
- **식사**: 인벤토리에 있는 음식을 먹는 행동. 즉석식품(전자레인지)은 직접, 배달 음식은 핸드폰으로 주문 후 인벤토리에 들어감. 불 사용 안 함 (귀신 설정과 일관)
- **핸드폰**: 내부 메뉴 3개 — 쇼핑, 배달, 시간 떼우기(쇼츠/영화/SNS). 검색·상담은 상시 메뉴 아님, 스토리 이벤트 시 연출로만 표시
- **인벤토리**: 1막부터 상시 접근 가능. 소지품(식료품, 생활용품, VR 콘텐츠 등) 확인. 3막부터 아이템 배치(제물) 기능 해금. 침대 머리맡 단상에 배치 → 귀신이 제물에 주의를 뺏긴 틈에 스킨십 가능
- **스킨십**: 귀신이 보일 때만 활성화 (3~4막)
- **공포 인터럽트**: 행동 중 랜덤으로 발생하는 공포 이벤트 (고개 돌리기 등). 메뉴가 아닌 연출. 안 보려면 키 연타/리듬으로 저항

### 2-4. 환경 피드백 (수치 비노출 대체)

#### 캐릭터 외형으로 전달
| 내부 상태 | 캐릭터 변화 |
|----------|-----------|
| 체력 좋음 → 저하 | 자세 변화: 등 펴고 서 있음 → 어깨 처짐 → 무릎 굽히고 상체 앞으로 → 벽에 기대/앉음 |
| 양기 정상 → 부족 | 피부톤 창백해짐 + 다크서클 진해짐. 심하면 눈 움푹, 핏줄 비침 |
| 공포 낮음 → 누적 | 표정: 무표정 → 두리번거림/어깨 움츠림 → 뭔가를 가리는 자세, 떨림 |

#### 배경 환경으로 전달
| 내부 상태 | 환경 변화 |
|----------|----------|
| 체력 저하 | 화면 채도 저하 — 바래진 느낌 |
| 양기 부족 | 화면 가장자리/상단에서 내려오는 어둠 (비네팅) |
| 공포 누적 | 간헐적 심장박동 같은 화면 떨림 + 화면 노이즈 (공포 영화 느낌) |
| 집 상태 나쁨 | 배경에 쓰레기/먼지 증가 |

## 3. 비주얼 노벨 화면 (VN)

> 대사, 선택지가 표시되는 스토리 연출 화면

### 3-1. 레이아웃
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [배경 CG / 일러스트]                      │
│                                                 │
│    [캐릭터 스탠딩]         [캐릭터 스탠딩]         │
│                                                 │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │ [화자명]                                 │    │
│  │                                         │    │
│  │  대사 텍스트가 여기에 표시됩니다.           │    │
│  │                                         │    │
│  │                    [Skip] [Auto] [▼]    │    │
│  └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
```

### 3-2. 대사창
- **위치**: 하단 20%
- **배경**: 반투명 검정 (알파 70%)
- **화자명**: 좌상단, 이름 박스
  - 주인공: 이름 미표시 (몰입용)
  - 귀신: "???" (이름 미상 유지)
  - 기타: 직함 또는 이름
- **텍스트**: 타이핑 연출 (클릭으로 즉시 표시 가능)
- **내면 독백**: 기울임체 + 약간 밝은 색상
- **스킵**: 기 읽은 텍스트 빠르게 넘기기 (Ctrl 홀드 또는 스킵 버튼)
- **오토**: 자동 진행 모드 (대사 표시 완료 후 일정 시간 뒤 다음으로). 대사창 우하단 토글

### 3-3. 선택지 (모달 방식)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [배경 + 캐릭터 블러 + 딤 처리]             │
│                                                 │
│           ┌─────────────────────┐               │
│           │                     │               │
│           │  선택지 A 텍스트     │               │
│           │                     │               │
│           │  선택지 B 텍스트     │               │
│           │                     │               │
│           │  선택지 C 텍스트     │               │
│           │                     │               │
│           └─────────────────────┘               │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **모달창 형태**로 화면 중앙에 표시
- 배경 블러 + 딤 처리 — 선택지 외 조작 불가
- 호버 시 하이라이트
- 시간 제한 선택지: 게이지 없이 **선택지 텍스트가 서서히 흐려짐**으로 표현

## 4. 아케이드 인터랙션 화면 (ARC)

> 공포/생존 인터랙션. 수면 중 또는 일상 중 랜덤 발생.

### 공통 실패 분기 (가위눌림 · 고개 돌리기 공통)
| 상태 | 결과 |
|------|------|
| 정신력/체력/양기 충분 | 도망 (가위눌림은 알람에 깸). 게임 계속 |
| 체력 높음 + 정신력 고갈 | 정신병 엔딩 |
| 체력 낮음 + 정신력 고갈 | 신체 강탈 → 자살 엔딩 |
| 체력 약함 | 심장마비 엔딩 |

### 4-1. 가위눌림 (수면 중 — 키 연타로 몸 풀기)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [1인칭 시점: 천장 + 어둠]                  │
│         멀리서 긁는 듯한 목소리의 중얼거림            │
│         귀신이 천천히 다가옴                        │
│                                                 │
│              몸을 움직일 수 없음                    │
│              좌/우 키 연타로 몸 풀기                │
│                                                 │
│         [실패 시: 귀신이 천장에서 얼굴로 접근]       │
│         [2차 기회: 이 과정에서도 연타로 탈출 가능]    │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **피해 누적형** — 한 번 실패해도 알람 소리에 깨어남 (즉사 아님)
- 귀신 얼굴을 볼수록 정신력 급락 + 체력 하락
- 코앞까지 오는 일을 3번 정도 겪으면 건강해도 사망권
- **키 안내**: 첫 발생 시에만 하단에 작은 힌트. 이후엔 표시 안 함

### 4-2. 고개 돌리기 (일상 중 — 방향키 연타로 저항)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [현재 시점에서 시선이 한쪽으로 끌림]          │
│         한쪽에서 "끄으으으" 불쾌한 소리              │
│                                                 │
│              시선이 소리 쪽으로 돌아감               │
│              반대 방향키 연타로 저항                 │
│                                                 │
│              보일 것 같을수록                       │
│              몸 떨림(화면 떨림) 강해짐               │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **즉시 판정형** — 실패하면 상태에 따라 즉시 분기
- 의도적으로 아무것도 보여주지 않는 연출 (보이지 않는 공포)
- SIM 행동 중 랜덤으로 끼어드는 인터럽트 이벤트
- 화면 떨림 + 심장박동 느낌으로 긴장감 표현

### 4-3. 젖 빨기 (3막 — 자는 척하며 혀 조작)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [클로즈업: 입 안 — 젖꼭지 + 혀]            │
│                                                 │
│              좌/우 방향키로 혀 좌우 왕복             │
│              누르는 속도 = 강도/속도                │
│                                                 │
│              피드백:                              │
│              귀신이 얼굴을 쓰다듬음 (성공)           │
│              손을 멈춤 (의심)                      │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **자는 척** — 들키지 않는 범위 내에서 최대한 혀를 움직이는 것이 목표
- 귀신이 얼굴을 쓰다듬어줌 — 의심스러우면 순간 손을 멈춤 (피드백)

| 입력 패턴 | 귀신 반응 | 결과 |
|----------|----------|------|
| 너무 빠름 | 의심 → 손 멈춤 → 사라짐 | 실패 |
| 너무 과격 | 알아챔 → 사라짐 | 실패 |
| 너무 느림/안 움직임 | 젖꼭지를 더 깊이 넣음 | 만족도 부족 |
| 눈치 못 채는 선에서 최대한 움직임 | 쓰다듬기 지속 + 편안한 표정 | 성공 → 젖꼭지 성감 개발 |

- **성공 보상**: 젖 분비량 증가 (시각) + 작고 기분 좋은 신음 (오디오)
- **귀신 표정**: 반복될수록 미세하게 만족/들뜬 표정으로 변화 (성적 만족인지 모성애 충족인지 애매해야 함)
- **2막 젖 물리기**: 주인공은 모름, 인터랙션 없음, 순수 VN 연출만. 반복 시 귀신 표정이 미세하게 변화하는 것만 표현

## 5. 성감 시뮬레이션 화면 (SEX)

> 앵커 기반 성감 개발 시뮬레이션. 배치 → 행동 선택 → 강도 조절 → 관찰 → 조정.

### 5-1. 레이아웃
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [캐릭터 CG: 귀신의 몸]                    │
│         (터치 가능 영역이 존재하지만               │
│          영역 표시는 하지 않음)                    │
│                                                 │
│    [앵커: 입]  [앵커: 왼손]                       │
│    [앵커: 오른손]  [앵커: 성기]                    │
│                                                 │
│              피드백:                              │
│              캐릭터의 신체 반응 + 표정 + 음성       │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 5-2. 앵커 시스템
- **4개 앵커**: 입, 왼손, 오른손, 성기
- **조작 흐름**: 부위에 배치 → 행동 선택 → 강도 조절 → 오토 시작
- **강도**: 속도+세기 통합 단일 값 (낮음 = 느리고 부드럽게 / 높음 = 빠르고 세게)
- **전부 오토 가능**: 4개 앵커 모두 오토로 두고, 반응을 보면서 조정
- **물리적 제약**: 체위에 따라 앵커 배치 가능 범위가 달라짐 (입과 성기가 물리적으로 불가능한 위치 조합 차단 등)

### 5-3. 행동 목록
| 앵커 | 행동 |
|------|------|
| **손 (바닥)** | 쓰다듬기, 움켜쥐기, 주무르기, 감싸기 |
| **손 (가락)** | 문지르기, 꼬집기, 잡아 비틀기, 잡아 당기기, 찌르기/넣기 |
| **입** | 가벼운 뽀뽀, 키스, 핥기, 빨기 |
| **성기** | 문지르기, 넣기(움직이기), 사정 |

- 모든 부위에 모든 행동 가능 (영역별 제한 없음)
- 부위 × 행동 조합마다 귀신의 반응이 다름

### 5-4. 부위별 성감 유형 (12종)

#### A. 순응형 (처음부터 긍정 반응)
| 세부 | 특성 |
|------|------|
| A1 미온 | 약하게 꾸준히 좋아함, 큰 변화 없음 |
| A2 성장 | 처음에 약하지만 점점 강해짐 |
| A3 포화 | 처음엔 좋지만 과하면 둔감해짐 |
| A4 역전 | 처음엔 좋지만 과하면 통증 → 거부 |

#### B. 개발형 (초기 거부 → 반복 시 긍정)
| 세부 | 특성 |
|------|------|
| B1 전환 | 싫어하다가 임계점 넘으면 확 좋아함 |
| B2 점진 | 아주 천천히 거부가 줄고 반응이 생김 |
| B3 폭발 | 오래 걸리지만 개발 완료 시 가장 강한 반응 |

#### C. 거부형 (지속적 부정 반응)
| 세부 | 특성 |
|------|------|
| C1 고정 | 항상 싫어함, 변하지 않음 |
| C2 악화 | 할수록 더 싫어함 |
| C3 트라우마 | 특정 조건(불, 통증 등)과 연결된 강한 거부 |

#### D. 무반응형
| 세부 | 특성 |
|------|------|
| D1 둔감 | 반응 없음, 개발 불가 |
| D2 잠재 | 평소 무반응이나 특정 조건 충족 시 반응 개시 |

### 5-5. 시각적 흥분 피드백 (게이지 대체)
| 단계 | 시각 변화 | 오디오 변화 |

| 절정 | 몸 경련, 표정 변화, 등 활처럼 휨 | 절정 음성 |

## 6. 오버레이 UI

### 6-1. 핸드폰 UI

핸드폰은 SIM 메뉴에서 접근하는 오버레이. 내부에 3개 앱이 있다.

```
┌─────────────────────────────────────────────────┐
│                                                 │
│          [배경: 방 (어두움)]                      │
│                                                 │
│                    ┌──────────┐                  │
│                    │          │                  │
│                    │ [쇼핑]   │                  │
│                    │ [배달]   │                  │
│                    │ [시간떼우기]│                 │
│                    │          │                  │
│                    └──────────┘                  │
│                  (핸드폰 프레임)                   │
│                                                 │
└─────────────────────────────────────────────────┘
```
- 화면 중앙~우측에 핸드폰 프레임 표시
- 배경은 어두운 방 — 핸드폰 화면 빛이 주인공 얼굴을 비춤
- **쇼핑**: 쇼핑 앱 UI. 카테고리별 목록 (생활용품, VR 콘텐츠, 제물 재료 등). 소지금 표시 (유일하게 수치가 직접 노출되는 항목). 구입한 물건은 인벤토리로.
- **배달**: 배달 앱 UI. 음식 주문 → 인벤토리에 들어감. 식사는 SIM 메뉴에서 별도로 수행.
- **시간 떼우기**: 쇼츠, 영화, SNS 등. 시간 소모 + 공포 소폭 감소.
- **검색/상담**: 상시 메뉴 아님. 스토리 이벤트 시 핸드폰 연출로만 표시.
- 핸드폰 닫기 = 오버레이 종료 (뒤로가기 또는 ESC)

## 7. 전환 효과

| 전환 | 효과 | 사용 상황 |
|------|------|----------|
| SIM → VN | 배경 유지, 행동 메뉴 페이드아웃 + 대사창 슬라이드업 | 이벤트 발생 시 |
| VN → SIM | 대사창 페이드아웃 + 행동 메뉴 슬라이드업 | 이벤트 종료 시 |
| SIM → ARC | 화면 급격히 어두워짐 + 1인칭 전환 | 공포 이벤트 돌입 |
| ARC → VN | 밝기 서서히 회복 + 대사창 등장 | 인터랙션 결과 연출 |
| VN → SEX | 배경 CG 전환 (크로스 디졸브) + 대사창 축소/소멸 | 성적 장면 돌입 |
| SEX → VN | CG 유지 + 대사창 재등장 | 대화/감정 연출 삽입 |
