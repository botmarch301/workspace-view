# 화면 구성 설계 (UI Layout)
> **Status**: 초안 | **Created**: 2026-03-29
> **참조**: [concept.md](concept.md) · [scenario.md](scenario.md) · [scenes.md](scenes.md)

---

## 설계 원칙

1. **수치 비노출** — 내부 수치(양기, 공포, 호감, 성욕, 흥분도)를 UI에 직접 표시하지 않는다
2. **몰입 우선** — HUD를 최소화하고, 필요한 정보는 환경/연출로 전달한다
3. **모드 전환 자연스럽게** — SIM ↔ VN ↔ ARC ↔ SEX 전환 시 UI가 부드럽게 변형된다

## 1. 공통 레이아웃

### 1-1. 해상도 및 비율
- **기본 해상도**: 1920×1080 (16:9)
- **세이프 존**: 좌우 상하 5% 마진
- **세로형 CG**: 캐릭터 스탠딩은 화면 중앙~우측 배치

### 1-2. 공통 UI 요소
| 요소 | 위치 | 설명 |
|------|------|------|
| **시스템 메뉴** | 우상단 (아이콘) | 세이브/로드/설정/종료. 평소 숨김, 마우스 호버 또는 ESC로 표시 |
| **퀵세이브** | 없음 | 의도적 생략 — 선택의 무게를 유지 |
| **오토세이브** | 매 행동 종료 시 | 슬롯 3개 로테이션 |

## 2. 일상 시뮬레이션 화면 (SIM)

> 하루의 행동을 선택하고 관리하는 메인 루프 화면

### 2-1. 레이아웃
```
┌─────────────────────────────────────────────────┐
│                                                 │
│              [배경: 집 뷰 A/B/C]                 │
│              (시간대에 따라 조명 변화)              │
│                                                 │
│                                                 │
│  ┌──────────┐                                   │
│  │ 시간 표시 │                                   │
│  │ ○월 ○일  │                                   │
│  │ [아침]   │                                   │
│  └──────────┘                                   │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │         행동 선택 메뉴 (하단)              │    │
│  │  [식사] [세신] [청소] [출근] [쇼핑]        │    │
│  │  [VR]  [수면] [조사] [제물] [대화]        │    │
│  └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
```

### 2-2. 시간 표시
- **위치**: 좌상단
- **표시**: 날짜 + 시간대 (아침/낮/저녁/밤/심야)
- **형식**: 달력 느낌 없이, 간결한 텍스트
- **월상(달 모양)**: 보름달이 가까워질수록 달 아이콘이 차오름 (3막 이후 표시)
  - 수치가 아닌 **시각적 변화**로 전달

### 2-3. 행동 선택 메뉴
- **위치**: 하단 중앙
- **표시 방식**: 아이콘 + 텍스트 버튼
- **비활성 표시**: 시간대/조건에 따라 일부 버튼 흐리게 (출근은 밤에 비활성 등)
- **해금 시스템**: 
  - 1막: 식사, 세신, 출근, 쇼핑, VR, 수면
  - 2막: + 인터넷 검색 (조사)
  - 3막: + 제물 놓기, 주변 대화
  - 4막: + 성감 개발, 대화, 유품 조사, 외출 조사

### 2-4. 환경 피드백 (수치 비노출 대체)
| 내부 상태 | 환경 변화 |
|----------|----------|
| 체력 저하 | 화면 채도 저하, 주인공 움직임 느려짐 |
| 양기 부족 | 주인공 외형 변화 (Phase 진행), 화면 가장자리 어두워짐 |
| 공포 누적 | 배경 그림자 증가, 간헐적 화면 떨림 |
| 집 상태 나쁨 | 배경에 쓰레기/먼지 증가 |
| 귀신 호감 높음 | 집이 깨끗해짐 (귀신의 호의), 간헐적 따뜻한 색조 |

## 3. 비주얼 노벨 화면 (VN)

> 대사, 회상, 선택지가 표시되는 스토리 연출 화면

### 3-1. 레이아웃
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [배경 CG / 일러스트]                      │
│                                                 │
│    [캐릭터 스탠딩]         [캐릭터 스탠딩]         │
│                                                 │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │ [화자명]                                 │    │
│  │                                         │    │
│  │  대사 텍스트가 여기에 표시됩니다.           │    │
│  │                                         │    │
│  │                              [▼ 다음]   │    │
│  └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
```

### 3-2. 대사창
- **위치**: 하단 20%
- **배경**: 반투명 검정 (알파 70%)
- **화자명**: 좌상단, 이름 박스
  - 주인공: 이름 미표시 (몰입용)
  - 귀신: "???" → 이름 발견 후 실명 표시
  - 기타: 직함 또는 이름
- **텍스트**: 타이핑 연출 (클릭으로 즉시 표시 가능)
- **내면 독백**: 기울임체 + 약간 밝은 색상

### 3-3. 선택지 (모달 방식)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [배경 + 캐릭터 블러 + 딤 처리]             │
│                                                 │
│           ┌─────────────────────┐               │
│           │                     │               │
│           │  선택지 A 텍스트     │               │
│           │                     │               │
│           │  선택지 B 텍스트     │               │
│           │                     │               │
│           │  선택지 C 텍스트     │               │
│           │                     │               │
│           └─────────────────────┘               │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **모달창 형태**로 화면 중앙에 표시
- 배경 블러 + 딤 처리 — 선택지 외 조작 불가
- 호버 시 하이라이트
- 시간 제한 선택지: 게이지 없이 **선택지 텍스트가 서서히 흐려짐**으로 표현

### 3-4. 회상 씬 (4막 귀신의 과거)
- **화면 처리**: 세피아 톤 + 비네팅 + 약간의 필름 그레인
- **대사창**: 평소보다 투명도 높임 (더 몽환적)
- **전환**: 물결 디졸브로 현재 ↔ 회상 구분

## 4. 아케이드 인터랙션 화면 (ARC)

> QTE, 리듬, 균형 조절 등 아케이드성 인터랙션

### 4-1. 가위눌림 (키 연타)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [1인칭 시점: 천장 + 어둠]                  │
│         (화면 가장자리에 검은 그림자 침식)           │
│                                                 │
│                                                 │
│                                                 │
│              시각적 피드백:                        │
│              화면 밝기로 저항 상태 표현              │
│              (밝아짐 = 저항 중 / 어두워짐 = 굴복)   │
│                                                 │
│         [키 입력 안내: 화면 하단 미세한 힌트]        │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **게이지 없음** — 화면 밝기/시야 범위로 상태 전달
- 연타 성공: 시야가 넓어지고 밝아짐 → 고개가 돌아감
- 연타 실패: 시야가 좁아지고 어두워짐 → 의식 잃음
- **키 안내**: 첫 발생 시에만 하단에 작은 힌트 ("연타하세요"). 이후엔 표시 안 함

### 4-2. 인기척 (균형 조절)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [1인칭 시점: 침대에 누운 채 천장]           │
│                                                 │
│              주인공의 시선이                        │
│              좌우로 흔들림                          │
│              (← 돌리고 싶지 않다 / 돌리고 싶다 →)   │
│                                                 │
│              카메라 흔들림 강도 =                   │
│              현재 균형 상태                         │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **좌우 키로 균형 조절** — UI 게이지 없이, 카메라 시선 흔들림으로 표현
- 너무 한쪽으로 치우침 → 고개가 확 돌아감 → 결과 분기

### 4-3. 젖 빨기 리듬 (자는 척)
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [클로즈업: 귀신의 가슴 + 주인공 얼굴]       │
│                                                 │
│              리듬 피드백:                          │
│              귀신의 호흡/표정 변화                   │
│              (편안함 = 성공 / 경계 = 실패)          │
│                                                 │
│              입력이 자연스러우면                     │
│              화면 색조가 따뜻해짐                    │
│                                                 │
└─────────────────────────────────────────────────┘
```
- **리듬 게이지 없음** — 귀신의 반응(표정, 호흡, 몸의 이완)으로 판단
- 너무 빠르거나 불규칙 → 귀신이 긴장/경계 표정 → 사라짐
- 적절한 리듬 → 귀신이 편안한 표정 → 계속 진행

## 5. 성감 시뮬레이션 화면 (SEX)

> 터치 기반 성감 개발 인터랙션

### 5-1. 레이아웃
```
┌─────────────────────────────────────────────────┐
│                                                 │
│         [캐릭터 CG: 귀신의 몸]                    │
│         (터치 가능 영역이 존재하지만               │
│          영역 표시는 하지 않음)                    │
│                                                 │
│              클릭/드래그로 터치                    │
│                                                 │
│              피드백:                              │
│              캐릭터의 신체 반응 + 표정 + 음성       │
│                                                 │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 5-2. 터치 시스템
- **입력**: 마우스 클릭, 드래그, 홀드
- **터치 영역**: 시각적 표시 없음 — 플레이어가 탐색해야 함
- **리듬 판정**: 입력 간격의 일관성과 속도를 내부적으로 판정
  - 느리고 일정한 리듬 = 높은 점수
  - 빠르거나 불규칙 = 낮은 점수

### 5-3. 시각적 흥분 피드백 (게이지 대체)
| 단계 | 시각 변화 | 오디오 변화 |
|------|----------|-----------|
| 초기 | 표정 미세 변화, 가슴 약간 부풀어오름 | 조용한 호흡 |
| 중반 | 젖꼭지 발기, 호흡 거칠어짐, 피부 상기 | 숨소리 커짐 |
| 후반 | 클리토리스/소음순 충혈, 애액 표현 | 신음, 호흡 불규칙 |
| 절정 | 몸 경련, 표정 변화, 등 활처럼 휨 | 절정 음성 |

### 5-4. 오토 모드
- 반복적 행위가 확립되면 **오토 전환 버튼** 표시 (화면 구석, 작게)
- 오토 중에는 다른 부위 탐색 가능

## 6. 오버레이 UI

### 6-1. 핸드폰 UI (검색 · 쇼핑 · 상담 공통)

검색, 쇼핑, 상담 등 정보 접근은 모두 **주인공의 핸드폰 화면**을 통해 이루어진다.

```
┌─────────────────────────────────────────────────┐
│                                                 │
│          [배경: 방 (어두움)]                      │
│                                                 │
│                    ┌──────────┐                  │
│                    │ ┌──────┐ │                  │
│                    │ │검색앱│ │                  │
│                    │ └──────┘ │                  │
│                    │          │                  │
│                    │ 검색결과 │                  │
│                    │ · 방화사건│                  │
│                    │ · 자살소문│                  │
│                    │ · 처녀귀신│                  │
│                    │          │                  │
│                    └──────────┘                  │
│                  (핸드폰 프레임)                   │
│                                                 │
└─────────────────────────────────────────────────┘
```
- 화면 중앙~우측에 핸드폰 프레임 표시
- 배경은 어두운 방 — 핸드폰 화면 빛이 주인공 얼굴을 비춤
- **검색**: 검색 앱 UI. 키워드 선택 방식 (직접 타이핑 아님)
- **쇼핑**: 쇼핑 앱 UI. 카테고리별 목록 (식량, 생활용품, VR 콘텐츠, 제물 재료)
  - 소지금 표시 (유일하게 수치가 직접 노출되는 항목)
- **상담**: 메신저/통화 앱 UI. 대화 형식으로 조언을 받음
- 핸드폰 닫기 = 오버레이 종료 (뒤로가기 또는 ESC)

### 6-2. 제물 놓기
- SIM 화면에서 아이템 선택 → 방 안에 배치
- 배치 후 시간 경과 → 귀신 반응 이벤트 트리거

## 7. 전환 효과

| 전환 | 효과 | 사용 상황 |
|------|------|----------|
| SIM → VN | 배경 유지, 행동 메뉴 페이드아웃 + 대사창 슬라이드업 | 이벤트 발생 시 |
| VN → SIM | 대사창 페이드아웃 + 행동 메뉴 슬라이드업 | 이벤트 종료 시 |
| SIM → ARC | 화면 급격히 어두워짐 + 1인칭 전환 | 공포 이벤트 돌입 |
| ARC → VN | 밝기 서서히 회복 + 대사창 등장 | 인터랙션 결과 연출 |
| VN → SEX | 배경 CG 전환 (크로스 디졸브) + 대사창 축소/소멸 | 성적 장면 돌입 |
| SEX → VN | CG 유지 + 대사창 재등장 | 대화/감정 연출 삽입 |
| 현재 → 회상 | 물결 디졸브 + 세피아 톤 전환 | 4막 과거 회상 |
| 회상 → 현재 | 역방향 물결 + 색상 복원 | 회상 종료 |
